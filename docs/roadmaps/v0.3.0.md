## Version goal

**Build the environment-independent foundation for real source clients, and deepen the
fixture-based Tier 0 loop — all without a live Magento, New Relic, or NerdGraph environment.**

v0.2.0 hardened the fixture-based loop inside Laravel. v0.3.0 lays the shared HTTP plumbing real
clients will sit on, wires credential-driven client selection with a clean fixture fallback,
adds narrow fixture-diagnosable Tier 0 patterns, and stands up contract-test scaffolding — so the
deferred real clients become a drop-in once credentials and a live environment are available.

## v0.3.0 success definition

The version is successful when the system can:

1. Resolve a provider-agnostic `HttpClient` seam from the container that performs no outbound
   calls until invoked, with transport failures mapped to `SourceFetchError` and rate limits honored.
2. Let a source adapter select its client from credential presence and fall back cleanly to the
   fixture client when credentials are absent — with no real calls in tests.
3. Run additional narrow, fixture-diagnosable Tier 0 diagnostic patterns.
4. Exercise a recorded-response contract-test harness using existing sample payloads (no live capture).
5. Document the HTTP seam, credential-driven fallback, and contract-test convention.

## Implementation rules

- Keep providers swappable: no vendor response shapes leak above the adapter seam — everything
  downstream still reasons over the canonical signal model.
- Reuse the v0.1.0/v0.2.0 seams (`SourceAdapter`, `RetryPolicy`, `PipelineObserver`, `SignalStore`,
  `SignalStoreFactory`); do not fork them.
- No live calls: the default `HttpClient` performs no outbound requests until invoked, and all
  tests use a fake/recorded transport. Do not add real provider calls or live capture.
- Stay narrow: do not build broad pattern-library depth, revenue attribution, economic-impact
  estimation, prediction, or external-market signals — those remain deferred north-star items.
- Every source-code phase is a reviewed PR with a `/code-review` pass (per `CLAUDE.md`).

## Phase 1 — HTTP source-client foundation

**Status:** [ ]

**Goal:** Establish the shared HTTP plumbing real source clients will need, behind clean
interfaces, without calling any real provider by default.

**Deliverables:**

- Define the `HttpClient` seam with request/response value objects and a timeout, auth-header,
  and JSON-handling contract.
- A default `HttpClient` implementation over Laravel's HTTP client that performs no outbound
  calls until invoked.
- A fake transport double implementing the seam for tests (no live calls).
- Transport/HTTP failures mapped to `SourceFetchError` with correct `retryable` classification
  (timeouts/5xx/429 retryable; 4xx auth/config not), so the existing `RetryPolicy` drives retries.
- Rate-limit awareness honoring `Retry-After`/429 as retryable.
- The `HttpClient` seam bound into the container and `config/provado.php` without enabling any
  real source.

---

## Phase 2 — Credential-driven adapter wiring (fixture fallback)

**Status:** [ ]

**Goal:** Let adapters choose a client from credential presence and fall back to the fixture
client when credentials are absent — provider-agnostic, with no real calls.

**Deliverables:**

- Credential-presence checks on `SourceConfig`/`SourceCredentials` so an adapter can decide which
  client to use.
- Client selection in the source adapters that falls back to the fixture client when credentials
  are absent, tested with a stub client (no live calls).

---

## Phase 3 — Deepen fixture-based diagnostic coverage

**Status:** [ ]

**Goal:** Broaden diagnostic coverage with narrow, deterministic patterns fully diagnosable from
Tier 0 fixture signals, plus targeted correlation and report polish — keeping the set narrow.

**Deliverables:**

- A 3DS / payment-config regression Tier 0 pattern (the v1 lead pattern), fixture-diagnosable,
  with full PHPUnit coverage.
- A catalog / feed sync-failure Tier 0 pattern over fixtures, with PHPUnit coverage.
- Time-proximity correlation that joins signals close in time alongside shared-entity joins,
  with tests. (Core correlation logic — plan approval required per `CLAUDE.md`.)
- Incident-report evidence ordering and de-duplication improvements, with tests.

---

## Phase 4 — Contract-test scaffolding and docs

**Status:** [ ]

**Goal:** Stand up the test scaffolding future real clients will need, and document the new
seams — all without live capture.

**Deliverables:**

- A recorded-response fixture loader and directory convention for future provider contract tests,
  reusing existing sample payloads (no live capture).
- `docs/ARCHITECTURE.md` updated to document the HTTP source-client seam, credential-driven
  fixture fallback, and contract-test convention.
- README/config docs updated for the new HTTP client options.
